（1）普通用户功能需求
普通用户是系统的主要使用群体，核心需求为完成健身打卡与个人记录管理，具体功能如下：
1.用户登录：输入用户名与密码进行身份验证，验证通过后进入个人中心页面；账号密码错误则提示登录失败，停留在登录页面。
2.提交健身打卡：登录成功后，用户可以选择对应的运动项目，填写打卡文字内容，提交打卡记录。系统自动补充打卡时间与默认审核状态，完成数据入库。
3.查看个人打卡记录：用户可以查看自己提交的全部打卡数据，页面展示运动项目、打卡内容、打卡时间、审核状态等完整信息，实时了解打卡审核结果。
（2）管理员功能需求
管理员负责系统整体数据管理与打卡审核工作，拥有最高操作权限，具体功能如下：
1.管理员登录：使用专属管理员账号密码登录系统后台，完成身份校验后进入管理页面。
2.查看全部打卡记录：管理员可以查看系统内所有用户的打卡记录，同时展示用户名、运动项目、打卡内容、时间、审核状态等信息，实现全局数据监控。
3.打卡审核：针对状态为 “待审核” 的打卡记录，管理员可以执行通过或驳回两种操作，操作完成后同步修改数据库中记录状态，实现线上审核流程。
