## 项目概要

- **类型**：分角色（普通用户 / 管理员）的健身打卡管理系统，演示性大作业。
- **技术栈**：Spring Boot + MyBatis + MySQL（后端），React + Tailwind + react-hot-toast（前端），RESTful API。
- **核心流程**：用户登录 → 选运动项目提交打卡（默认待审核）→ 管理员审核（通过/驳回）→ 用户查看审核结果。
- **预留能力**：AI 健身建议对话（先设计接口，主体功能完成后实现）。
- **运行方式**：Mac 开发，Windows 演示，原生运行（JDK + MySQL + 浏览器），不使用 Docker / Electron。

## 文档约定

- 所有设计为**演示项目级别**，在保证作业要求（分层、外键、RESTful、统一异常）的前提下取最简实现。
- 文档中标注「待确认」「权衡」「预留」的部分，实现时若有调整，需同步更新对应文档，并在根目录 [`../AGENTS.md`](../AGENTS.md) 记录。
