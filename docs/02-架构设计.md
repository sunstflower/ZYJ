# 架构设计

> 项目：健身打卡系统（JSA）
> 文档版本：v1.0
> 最后更新：2026-06-16

## 1. 技术选型

| 层 | 技术 | 说明 |
|----|------|------|
| 前端 | React 18 + Vite | SPA 单页应用，构建快 |
| 前端样式 | Tailwind CSS | 原子化 CSS |
| 前端提示 | react-hot-toast | 操作提示（成功/失败 Toast），作业指定 |
| 前端请求 | axios | 封装统一请求/响应拦截器 |
| 前端路由 | react-router-dom | 登录页 / 用户中心 / 管理后台 |
| 后端 | Spring Boot 3.3.x（JDK 17） | RESTful 后端 |
| 持久层 | MyBatis 3 + mybatis-spring-boot-starter | DAO 层 |
| 数据库 | MySQL 8.x | utf8mb4 |
| 构建 | 后端 Maven，前端 npm | |

> **JDK 版本说明**：Spring Boot 3.x 要求 JDK 17+。若演示环境只有 JDK 8，则需改用 Spring Boot 2.7.x，文档与依赖需相应调整。**实现前务必确认 Windows 演示机的 JDK 版本，并记录到 AGENTS.md。**

## 3. 后端分层设计（Controller / Service / DAO）

严格三层分离，依赖方向自上而下：

| 层 | 职责 | 不应做 |
|----|------|--------|
| Controller | 接收 HTTP 请求、参数绑定与基础校验、调用 Service、封装统一响应 | 不写业务逻辑、不直接访问数据库 |
| Service | 业务逻辑、状态流转规则、事务控制、组合多个 DAO | 不感知 HTTP（不依赖 Request/Response） |
| DAO（Mapper） | 数据库 CRUD，MyBatis SQL | 不写业务判断 |

## 4. 统一响应体

所有接口返回统一 JSON 结构：

```json
{
  "code": 200,
  "message": "success",
  "data": { }
}
```

- `code`：业务状态码（见 [接口文档](./04-接口文档.md) 错误码约定），与 HTTP 状态码配合使用。
- `message`：提示信息，前端可直接用于 toast 展示。
- `data`：业务数据，可为对象、数组或 null。

```java
// Result.java（示意）
public class Result<T> {
    private int code;
    private String message;
    private T data;
    public static <T> Result<T> success(T data) { ... }
    public static Result<Void> error(int code, String message) { ... }
}
```

## 5. 统一异常处理

使用 `@RestControllerAdvice` + `@ExceptionHandler` 全局捕获：

| 异常类型 | 处理 |
|----------|------|
| `BusinessException`（自定义） | 返回对应业务 code 与 message，HTTP 400/401/403 视情况 |
| 参数校验异常（`MethodArgumentNotValidException`） | 返回 400 与字段错误信息 |
| 其它未捕获异常（`Exception`） | 返回 500，记录日志，对外返回通用错误信息 |

业务代码中通过 `throw new BusinessException(ResultCode.LOGIN_FAILED)` 抛出，由全局处理器统一转为 `Result`，Controller 不再写 try-catch。

## 6. 认证与鉴权方案（演示级）

> 演示项目采用轻量方案，重点是讲清"登录态 + 角色权限"，而非生产级安全。

1. **登录**：`POST /api/auth/login` 校验用户名密码，成功后下发 `token`（演示可用 UUID 或简单签名串），同时返回用户基本信息与 `role`。
2. **token 存储**：前端存 localStorage；axios 请求拦截器自动在 `Authorization` 头携带。
3. **服务端鉴权**：用拦截器（`HandlerInterceptor`）校验 token；需要管理员权限的接口额外校验 `role == ADMIN`。
4. **token→用户** 的映射：演示阶段可用内存 `Map<token, userId>`（重启失效，够用）；如需持久可后续换 JWT（自包含、无状态）。

> **设计权衡**：内存 token 实现简单、易讲解，但重启或多实例会失效。本项目为单机演示，采用内存方案；若要求更"正规"，再升级 JWT。该决策记录于 AGENTS.md。

## 7. 跨域（CORS）

前端开发态（Vite，端口 5173）与后端（8080）分离，需配置 CORS。两种方式择一：
- **开发**：Vite `server.proxy` 代理 `/api` 到后端，规避跨域。
- **后端**：全局 CORS 配置允许前端来源。

演示打包阶段，建议前端 `npm run build` 后将 `dist/` 静态资源放入后端 `resources/static/`，由 Spring Boot 同源托管，**彻底规避跨域**，且 Windows 上只需 `java -jar` 一条命令。

## 8. 跨平台与部署（Mac 开发 → Windows 演示）

Java/React 技术栈天然跨平台，关键约定：

1. **后端产物**：`mvn package` 生成可执行 `jar`，在 Windows 上 `java -jar jsa.jar` 即可运行（依赖 JVM，与系统无关）。
2. **前端产物**：`npm run build` 生成纯静态文件，无平台差异；推荐合并进后端 jar 同源托管。
3. **数据库**：Windows 安装 MySQL，执行 `init.sql` 重建数据与测试数据。
4. **配置外置**：MySQL 用户名/密码/端口写在 `application.yml`，演示机不同时只改配置不改代码。
5. **编码与时区**：JDBC URL 加 `?useUnicode=true&characterEncoding=utf8&serverTimezone=Asia/Shanghai`。
6. **换行符**：仓库配 `.gitattributes`（`* text=auto`）避免 CRLF/LF 混乱。

> 演示**不使用 Docker、不使用 Electron**：本系统是标准 B/S Web 应用，原生运行（JDK + MySQL + 浏览器）最简单、最不易在演示现场出错。详见根目录关于部署方式的讨论。

## 9. 演示启动流程（Windows）

```
1. 启动 MySQL，执行 init.sql 建库建表+测试数据
2. 在 application.yml 配置好数据库连接
3. java -jar jsa.jar          → 控制台出现 "Started JsaApplication on port 8080"
4. 浏览器访问 http://localhost:8080
5. 用 admin / 123456 演示管理端；alice / 123456 演示用户端
```
