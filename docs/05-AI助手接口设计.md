# AI 健身建议助手 — 接口预留设计

> 项目：健身打卡系统（JSA）
> 状态：**仅设计，暂不实现**。待主体功能（登录、打卡、审核）完成且跑通后再实现。
> 文档版本：v1.0
> 最后更新：2026-06-16

## 1. 目标

为健身用户提供一个对话式 AI，根据用户的提问（以及可选的打卡历史）给出个性化健身建议。本功能复杂度较高，因此**先定接口契约**，让前后端可以并行预留位置，主体功能完成后再接入真实大模型 API。

## 2. 设计原则

1. **后端代理调用**：前端 → 本系统后端 → 外部 AI 服务。**绝不**在前端直接持有 AI 服务的 API Key（防泄露）。Key 配置在后端 `application.yml` 或环境变量。
2. **接口与厂商解耦**：Service 层定义 `AiAdviceService` 接口，具体实现可对接任意大模型（Claude / OpenAI 兼容 / 国内厂商等）。先提供一个 `MockAiAdviceService` 返回假数据，保证主流程不依赖外部服务即可演示。
3. **可降级**：外部服务不可用时，返回友好提示而非 500，不影响系统其它功能。

## 3. 接口契约

### 3.1 获取 AI 健身建议

`POST /api/ai/advice`  权限：`USER`（登录用户）

**请求体**

```json
{
  "question": "我最近想增肌，每周练 3 次，怎么安排？",
  "includeHistory": false
}
```

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| question | string | 是 | 用户的问题/诉求 |
| includeHistory | boolean | 否 | 是否结合本人近期打卡记录作为上下文（默认 false） |

**成功响应** `200`

```json
{
  "code": 200,
  "message": "success",
  "data": {
    "answer": "建议采用分化训练……（AI 生成的建议文本）",
    "model": "mock",
    "createdAt": "2026-06-16 10:00:00"
  }
}
```

**降级/失败响应** `200`（带降级标识）或 `503`

```json
{
  "code": 200,
  "message": "AI 服务暂不可用，以下为通用建议",
  "data": {
    "answer": "保证充足睡眠与蛋白质摄入，循序渐进增加训练量。",
    "model": "fallback",
    "createdAt": "2026-06-16 10:00:00"
  }
}
```

> **可选扩展（流式）**：如需打字机效果，后续可增加 `GET /api/ai/advice/stream`（SSE，`text/event-stream`）。一期不做，先用一次性返回。

## 4. 后端设计预留

```
com.jsa
├── controller/AiController.java       # POST /api/ai/advice
├── service/AiAdviceService.java       # 接口：String advise(Long userId, AiAdviceRequest req)
└── service/impl
    ├── MockAiAdviceService.java        # 一期：返回固定/模板建议，便于演示
    └── RealAiAdviceService.java        # 二期：真正调用外部大模型 API
```

**配置项预留**（`application.yml`）：

```yaml
ai:
  enabled: false              # 一期 false，走 Mock
  provider: claude            # claude / openai-compatible / ...
  api-key: ${AI_API_KEY:}     # 从环境变量注入，勿写入仓库
  base-url: https://api.anthropic.com
  model: claude-haiku-4-5
  timeout-ms: 15000
```

> 通过 `@ConditionalOnProperty(name = "ai.enabled", havingValue = "true")` 或在工厂中按配置选择实现类，实现 Mock 与真实服务的切换。

## 5. 上下文构造（includeHistory=true 时）

当用户允许结合历史时，后端从 `checkin_record` 取该用户近期（如最近 10 条）已通过的打卡，拼成简要上下文一并发给模型，例如：

```
用户近期健身记录：
- 2026-06-15 跑步：晨跑5公里
- 2026-06-13 力量训练：练胸和背
请基于以上记录回答用户问题：<question>
```

注意：仅传必要的、用户本人的数据，不要把他人记录或敏感信息发给外部服务。

## 6. 实现里程碑（建议顺序）

1. ☐ 一期：定义接口 + `AiController` + `MockAiAdviceService`，前端接好对话 UI，端到端跑通（返回假数据）。
2. ☐ 二期：实现 `RealAiAdviceService`，接入真实大模型 API，做超时与降级处理。
3. ☐ 三期（可选）：`includeHistory` 上下文增强、SSE 流式输出。
